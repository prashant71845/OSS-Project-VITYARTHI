## Project Description
This project is an audit and analysis of open source software using the Linux Kernel as the primary example. The aim of this project is to understand how open source systems are designed, developed, and maintained, along with the philosophy behind free and open software.

The Linux Kernel was chosen because it forms the core of many operating systems and is one of the most widely used open source projects in the world. It demonstrates how large-scale collaboration can lead to powerful and reliable software.

Along with theoretical analysis, this project also includes five shell scripts that demonstrate practical Linux concepts such as system information retrieval, file and directory handling, permission checking, log file analysis, and basic automation.

This combination of theory and hands-on scripting helps in understanding both the conceptual and practical aspects of open source software in a real-world environment.


## Scripts Overview

### Script 1: System Identity Report
Displays system information such as user, kernel version, date, and uptime.

### Script 2: FOSS Package Inspector
Checks if a package is installed and displays its version and purpose.

### Script 3: Disk and Permission Auditor
Analyzes directories and shows their permissions and size.

### Script 4: Log File Analyzer
Reads a log file and counts occurrences of a keyword.

### Script 5: Open Source Manifesto Generator
Takes user input and generates a personalized open source statement.
